    CSS Transform Traps Fixed Positioning — Use Portal

    Date: 2026-04-09 Source: TCONSIAM ERP Modal bug Confidence: High (verified fix)

    Pattern

    When a parent element has CSS `transform` (even from animations like `fadeIn` with `translateY`), it creates a new containing block for `position: fixed` descendants. This means the "fixed" element is no longer relative to the viewport — it's trapped inside the parent.

    Symptoms

  • Modal appears inside content area instead of full-screen overlay
  • User reports "stuck in small box" or needs to scroll within modal
  • `position: fixed` + `inset: 0` doesn't cover viewport

    • Fix

    Use React `createPortal(jsx, document.body)` to render modals at document root, bypassing any parent transform/filter/will-change containing blocks.

    Also Triggers On

  • `filter` property on parent
  • `will-change: transform` on parent
  • `backdrop-filter` on parent
  • `contain: paint` on parent

    • TCONSIAM Context

  • `animate-fade-in` uses `transform: translateY(8px)` → trapped Modal
  • Fixed in `ui/src/components/ui/Modal.tsx` with `createPortal`
  • All modals across 17+ modules benefited from this single fix
