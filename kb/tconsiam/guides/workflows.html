Development Workflows Guide

1. Development Setup

Start the development environment with hot-reload enabled on both API and UI services.

Quick Start

cd /root/projects/tcon
cd api && bun run dev          # API on :3457
cd ui && bunx vite             # UI on :5173 (proxies /api)
Note: The UI automatically proxies /api requests to localhost:3457. Open your browser to http://localhost:5173

What's Running

  • API Bun server on port 3457 — REST endpoints, auth, database
  • UI Vite dev server on port 5173 — React app with hot reload
  • Database SQLite at ~/.config/tconsiam/db/tconsiam.db

2. Deploy to Production

Follow this process to safely deploy changes to the production server (76.13.221.42:8890).

Deployment Command

cd /root/projects/tcon
./deploy.sh

What the script does:

  1. Build the UI: bunx vite build
  2. Copy dist/ → /opt/tconsiam-dashboard/ (full replacement)
  3. Restart tconsiam-api.service
  4. Verify health checks
Important: Always build the UI first before deploying. The deployment assumes a fresh dist/ folder.

Post-Deployment Checks

# View API logs
journalctl -u tconsiam-api -f

# Test health endpoint
curl https://tconsiam.tcon-siam.com/api/health

# Check database sync
ssh root@76.13.221.42 sqlite3 ~/.config/tconsiam/db/tconsiam.db ".tables"

3. Adding a New Module (Step by Step)

Modules are self-contained features that manage a specific domain (Products, Inventory, Invoices, etc).

  1. Define database schema

    Edit api/src/db/schema.ts — add your tables with proper relationships and constraints.

  2. Create migration

    Add SQL to api/src/db/migrate.ts with the next version number (e.g., v018 if latest is v017).

  3. Create API route file

    Create api/src/api/module.ts with endpoints (GET, POST, PUT, DELETE). Always wrap responses in a named key: { products: [...] }

  4. Mount in API index

    Import and register in api/src/index.ts — add to Hono app with app.route('/api/module', moduleRouter)

  5. Create UI component

    Create ui/src/components/ModuleView.tsx — React component with fetch, state management, and UI.

  6. Register in App routing

    Add to ui/src/App.tsx views array and ui/src/components/Layout.tsx NAV_ITEMS for sidebar navigation.

Pro tip: Start with a simple module (e.g., Categories) to learn the pattern, then expand to complex modules with relationships.

4. Key Patterns

API Response Format
Always wrap results in a named key
// Correct
{ products: [{id: 1, name: "Solar Panel"}, ...] }

// Wrong — bare arrays cause parsing issues
[{id: 1, name: "Solar Panel"}, ...]
Authentication
JWT Bearer header with role-based access control
// API request
fetch('/api/products', {
  headers: { 'Authorization': `Bearer ${token}` }
})

// Roles: boss (full access) | warehouse (limited by permissions)
Audit Logging
Track all changes for compliance and debugging
logAudit({
  action: 'UPDATE',
  tableName: 'products',
  recordId: 42,
  oldValues: { name: 'Panel A', price: 100 },
  newValues: { name: 'Panel B', price: 120 },
  userId: 'user123',
  userName: 'John Doe'
})
Stock Calculation
Stock is computed in real-time from inventory movements
SELECT product_id, SUM(quantity) as current_stock
FROM inventory_movements
GROUP BY product_id

// No separate stock table — movements are the source of truth
PDF Generation
PDFKit with Sarabun font, use lineBreak: false
const doc = new PDFDocument();
doc.font('/path/to/Sarabun-Regular.ttf');
doc.fontSize(12).text('Invoice', { lineBreak: false });

// Reset y before footer to avoid overlap
doc.y = doc.page.height - 80;
doc.text('Page footer', { align: 'center' });
Date Format
Store as ISO 8601 in SQLite, display in Thai locale on UI
// Storage
INSERT INTO invoices (date) VALUES ('2026-04-09T14:30:00Z')

// Display (React)
new Date('2026-04-09T14:30:00Z')
  .toLocaleDateString('th-TH', { year: 'numeric', month: 'long', day: 'numeric' })

5. Common Pitfalls

Wrong Repository Path

Repo is at /root/projects/tcon NOT tecon (missing the 's').

# Right
cd /root/projects/tcon

# Wrong
cd /root/projects/tecon  # Does not exist
PDFKit Footer Overlap

Always reset doc.y before writing the footer to prevent text overlap.

// Without this, footer overwrites content above
doc.y = doc.page.height - 80;
doc.text('Footer text', { align: 'center' });
Thai Characters in PDF

PDFKit garbles Thai text. Keep labels in English, use Thai only in data fields where unavoidable.

// Good
doc.text('Invoice Number: INV-2026-001');
doc.text('วันที่: 9 เมษายน 2026');

// Bad — Thai labels in PDFKit look corrupted
doc.text('เลขที่ใบแจ้งหนี้: INV-2026-001');
API Field Name Mismatch

API returns totalAmount and vat, but UI might expect subtotal and vatRate. Check the schema first.

// Verify in api/src/db/schema.ts
export const invoices = sqliteTable('invoices', {
  totalAmount: real().notNull(),
  vat: real().default(0)
});
Deploy Sequence Error

Must build UI first, then copy dist/. Deploying without a fresh build leaves old files in production.

# Right
bunx vite build
cp -r dist/ /opt/tconsiam-dashboard/
systemctl restart tconsiam-api.service

# Wrong — mixing old and new files
cp /root/projects/tcon/dist/ /opt/tconsiam-dashboard/
# (if dist/ is stale, production serves old code)
Port Confusion

Production uses port 8890 via Nginx. Direct API calls are port 3457. Dev is 5173.

Dev:   http://localhost:5173 → proxies /api to :3457
Prod:  https://tconsiam.tcon-siam.com:8890 → Nginx
API:   localhost:3457 (direct, dev/local only)

6. Infrastructure

Component Details
VPS 76.13.221.42 — Primary production server
API Server Bun on port 3457, managed by systemd (tconsiam-api.service)

7. UI Architecture

Tech Stack

  • React 19 + Vite — Modern component framework with hot reload
  • Tailwind CSS 4 — Utility-first styling with design tokens
  • Zustand — Lightweight state management for auth and app state
  • lucide-react — Icon library (~200 icons)
  • Routing — Hash-based (#/) for simple SPA navigation

Key Concepts

apiFetch() wrapper

Automatically adds JWT Bearer header to all requests. No manual auth handling needed.

// Automatically includes Authorization header
const data = await apiFetch('/api/products');
Shared Components

7 reusable UI components: Button, Modal, Field, Badge, Spinner, PageHeader, ResponsiveTable

CSS Design Tokens

Consistent theming via Tailwind config — dark background, amber accents, blue code highlights

Mobile Optimization

14px baseline font, 44px touch targets, responsive grid layouts, tested on iOS 14+ and Android 8+
