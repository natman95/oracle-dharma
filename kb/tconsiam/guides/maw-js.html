📋 Overview

MAW-JS is a powerful multi-agent orchestration system built with Bun + Hono + React 19 + Three.js + xterm.js. It provides unified control for managing Oracle AI agents across single machines and federated networks.

Core Capabilities

  • 58+ CLI commands organized by domain (agent lifecycle, fleet, communication, consciousness, federation)
  • Multi-transport routing with automatic failover (tmux → MQTT → HTTP)
  • Federation mesh for cross-machine communication with HMAC-SHA256 authentication
  • Hybrid status detection using command names, screen hashes, and real hooks for crash detection
  • SQLite + Drizzle ORM with WAL mode for persistence and auto-migrations
  • Web Dashboard featuring real-time agent monitoring, PTY streaming, and consciousness visualization
  • Discord integration with bidirectional chat, deploy alerts, and health escalation

✨ Key Features

Multi-Transport Routing

Intelligent routing with automatic failover ensures reliable communication:

  • Tmux (Local) — Direct screen attachment for same-machine agents (lowest latency)
  • MQTT (Cross-Host) — Publish-subscribe for federation mesh and remote agents
  • HTTP (Fallback) — Stateless REST API endpoint as final fallback

Federation Mesh

Cross-machine communication with security:

  • HMAC-SHA256 authentication for peer verification
  • Automatic peer discovery and routing
  • Health monitoring and connection status tracking
  • Peer routing optimization (findWindow null fix in v1.8.0)

Hybrid Status Detection

Combines multiple signals to accurately detect agent state:

  • Command name inspection
  • Screen hash validation (unique identifier per session)
  • Real hooks for crash detection
  • Fallback to last-known state

Web Dashboard

Real-time monitoring and control:

  • Live agent status and metrics
  • PTY (pseudo-terminal) streaming for remote shell access
  • 3D consciousness visualization via Three.js
  • Multi-user support with WebSocket sync

Discord Integration

Bidirectional communication bridge:

  • Chat with agents via Discord channels
  • Deployment alerts and notifications
  • Health escalation for critical issues

Data Persistence

Robust database layer:

  • SQLite with WAL (Write-Ahead Logging) mode for crash safety
  • Drizzle ORM for type-safe database access
  • Auto-migrations on startup

🖥️ CLI Usage Examples

List All Agents

maw ls

Wake an Agent

maw wake oracle-name

Create New Agent

maw bud --name my-agent

Peer-to-Peer Communication

maw talk-to agent-name "message content"

Broadcast to Fleet

maw broadcast "message for all agents"

Fleet Synchronization

maw fleet sync

Soul Sync with Project Scope

maw soul-sync --project my-project

Federation Status

maw federation status
